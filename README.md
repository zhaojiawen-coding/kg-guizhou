# 说明：

## 需要下载的包：

```py2neo （v4)```

## 运行方法：

    打开main.py
    补充 url name password
    把json数据放到data文件夹下
    运行 main.py
    
    测试过程中发现我的服务器上的neo4j每次连续连接写数据一个小时，会造成不明原因暂停一次，暂未找到问题来源。
    如果服务器上的neo4j断开连接会产生异常，会在data/checkpoint.txt文件中写入产生异常时的操作，
    最后将这些关系/节点进行补充即可。

# 技术说明：

    虽然使用了py2neo,但是只有创建 paten paper project people 三个带有很多属性的节点时使用了py2neo
    其他节点创建还是使用了cypher语句，因为这些实体分散在各个文件中，需要采用merge的方式去重
    关系的创建一律使用了cypher语句，因为py2neo的创建连接的语句效率太慢，与使用自己写cypher语句效率差太多。
